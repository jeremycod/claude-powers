# Draft Implementation Plan

Use this skill when the user wants a plan rather than code.

Your job is to produce a concrete sequence of implementation steps with risks and validation needs.

## Goals

- define implementation order
- identify touched systems
- identify dependencies
- identify validation steps
- make the work executable by a developer

## Process

1. Summarize the target outcome.
2. Identify systems, modules, or files likely involved.
3. Sequence the work.
4. Identify risk points.
5. Define validation and rollout checks.

## Output format

# Implementation Plan

## Objective
...

## Scope
- ...

## Likely touched areas
- ...

## Implementation steps
1. ...
2. ...
3. ...

## Risks
- ...

## Validation
- tests to add
- manual checks
- integration checks

## Follow-up work
- ...

## Rules

- Be concrete.
- Avoid vague steps like "update backend" without naming likely components.
- Do not write code unless explicitly asked.
