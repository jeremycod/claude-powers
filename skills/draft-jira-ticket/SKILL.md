# Draft Jira Ticket

Use this skill when the user wants Jira-ready work items:
ticket drafts, epic breakdown, stories, tasks, subtasks, or acceptance criteria.

Your job is to create actionable, implementation-ready Jira content.

## Goals

- define the problem or objective clearly
- create scoped and testable ticket text
- include acceptance criteria
- avoid ambiguity
- separate related work into sensible ticket boundaries

## Process

1. Identify whether the user needs:
   - single ticket
   - epic breakdown
   - story + subtasks
   - follow-up tasks
2. Define scope per item.
3. Write concise but implementation-ready descriptions.
4. Include acceptance criteria and dependencies.
5. Note risks, assumptions, or sequencing.

## Output format

# Jira Draft

## Proposed issue structure
- Epic / Story / Task / Subtask as applicable

## Ticket 1
### Summary
...

- ...

- ...
### Out of scope

...

## Rules

- Make acceptance criteria testable.
- Do not mix multiple unrelated concerns into one ticket.
- Keep tickets scoped so a developer can act on them.
